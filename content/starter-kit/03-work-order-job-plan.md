#### Fase 1: Request (Aanvraag)

**Wie:** Operator, supervisor, of automatisch via CMMS alarm

**Wat te doen:**
- Vul header informatie volledig in
- Beschrijf probleem zo specifiek mogelijk
- Bepaal juiste prioriteit (zie criteria)
- Selecteer juiste work type

**Prioriteit bepalen:**

| Priority | Criteria | Response Time |
|----------|----------|---------------|
| **Emergency** | Veiligheidsrisico, milieu-incident, complete productiestop | < 4 uur |
| **Urgent** | Significante impact op productie, snel verslechterende conditie | < 24 uur |
| **Normal** | Productie-impact beperkt, kan gepland worden | < 1 week |
| **Low** | Geen directe impact, kan wachten op convenient moment | Gepland |

---

#### Fase 2: Planning

**Wie:** Planner of maintenance supervisor

**Wat te doen:**
- Link juiste Job Plan (of maak nieuwe)
- Bepaal benodigde skills en uren
- Check en reserveer parts
- Identificeer benodigde permits
- Schat doorlooptijd

**Checklist:**
- â˜ Is Job Plan beschikbaar en actueel?
- â˜ Zijn alle parts in stock?
- â˜ Zijn de juiste mensen beschikbaar?
- â˜ Zijn er conflicten met productie-schedule?
- â˜ Welke permits zijn nodig?

---

#### Fase 3: Scheduled (Ingepland)

**Wie:** Planner in overleg met productie

**Wat te doen:**
- Bepaal exacte start datum/tijd
- Assign aan technician(s)
- Communiceer naar alle stakeholders
- Zorg dat permits klaar zijn

---

#### Fase 4: Execution (Uitvoering)

**Wie:** Technician(s)

**Wat te doen:**
- Volg Job Plan stap voor stap
- Documenteer TIJDENS het werk (niet achteraf!)
- Registreer afwijkingen en extra bevindingen
- Log alle gebruikte parts
- Registreer alle uren

**Tips voor goede documentatie:**
- Maak foto's van before/after
- Noteer serienummers van vervangen components
- Beschrijf wat je ziet, niet wat je denkt
- Als je afwijkt van Job Plan, leg uit waarom

---

#### Fase 5: Closed (Afgesloten)

**Wie:** Supervisor + technician

**Wat te doen:**
- Review completeness van documentatie
- Bepaal root cause en failure mode
- Schrijf aanbevelingen
- Verzamel sign-offs
- Sluit formeel af in systeem

### Kwaliteitscriteria voor een Goede Work Order

Een "goede" work order scoort op deze 10 punten:

| # | Criterium | âœ“/âœ— |
|---|-----------|-----|
| 1 | **Complete header** - Alle velden ingevuld, juiste asset gelinkt | |
| 2 | **Duidelijke probleem beschrijving** - Iedereen begrijpt wat er aan de hand is | |
| 3 | **Juiste prioriteit** - Onderbouwd, niet alles is "urgent" | |
| 4 | **Correcte classificatie** - Work type en maintenance type kloppen | |
| 5 | **Realistische planning** - Uren, skills en timing zijn haalbaar | |
| 6 | **Parts gedocumenteerd** - Alles wat gebruikt is staat erin | |
| 7 | **Uren correct** - Werkelijke uren, niet geschat | |
| 8 | **Root cause bepaald** - Niet alleen "fixed", maar ook "waarom kapot" | |
| 9 | **Failure mode ingevuld** - Hoe faalde het component | |
| 10 | **Aanbevelingen gedaan** - Wat kunnen we leren/verbeteren | |

**Score interpretatie:**
- 9-10: Excellent - Dit is het doel
- 7-8: Good - Acceptabel
- 5-6: Fair - Verbetering nodig
- <5: Poor - Training/coaching nodig

### Common Mistakes to Avoid

#### âŒ Bij Work Order Creation

| Fout | Waarom problematisch | Hoe te voorkomen |
|------|---------------------|------------------|
| "Pomp kapot" als beschrijving | Te vaag, technician weet niet wat te verwachten | Beschrijf symptomen: "Pomp P-101 maakt maalgend geluid, flow 30% onder normaal" |
| Verkeerd asset gelinkt | History bouwt op bij verkeerde asset | Dubbelcheck asset tag fysiek |
| Alles is "Emergency" | Echte emergencies krijgen geen prioriteit | Gebruik prioriteit criteria |
| Work order na afloop maken | Timing en details zijn onnauwkeurig | Maak WO VOOR je begint |

#### âŒ Bij Planning

| Fout | Waarom problematisch | Hoe te voorkomen |
|------|---------------------|------------------|
| Uren te laag schatten | Altijd tijdsdruk, ontevreden klanten | Basis op historical data, niet op hoop |
| Parts niet checken | Werk begint, parts niet beschikbaar | Altijd vooraf stockcheck |
| Geen Job Plan gebruiken | Inconsistente kwaliteit | Standaardiseer terugkerend werk |

#### âŒ Bij Execution

| Fout | Waarom problematisch | Hoe te voorkomen |
|------|---------------------|------------------|
| Parts niet registreren | Stock klopt niet, kosten onbekend | Scan of noteer ALLES |
| "Gerepareerd" zonder details | Geen leermoment, geen historie | Beschrijf wat je deed en zag |
| Achteraf documenteren | Details worden vergeten | Real-time of direct na afronding |
| Job Plan niet volgen | Stappen overgeslagen, kwaliteitsproblemen | Gebruik als checklist, vink af |

#### âŒ Bij Closing

| Fout | Waarom problematisch | Hoe te voorkomen |
|------|---------------------|------------------|
| Root cause = "Unknown" (te vaak) | Geen verbetering mogelijk | Doe 5-Why, vraag hulp |
| Geen recommendations | Zelfde probleem komt terug | Altijd nadenken over preventie |
| WO open laten staan | Backlog vervuild, KPIs kloppen niet | Review process met supervisor |
