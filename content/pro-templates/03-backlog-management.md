- [ ] Generate current backlog report from CMMS
- [ ] Pull aging report (WOs > 30 days)

- [ ] Identify duplicate/similar WOs

- [ ] Review backlog weeks trend (improving/declining?)

- [ ] Update CMMS with all changes
- [ ] Send meeting notes to attendees
- [ ] Create action items with owners/due dates
- [ ] Escalate blocked WOs to management

## 4. Priority Re-assessment Criteria

### When to Re-assess Priority

| Trigger | Action Required |
|---------|-----------------|
| WO age exceeds SLA by 2x | Mandatory review |
| Equipment condition changed | Re-evaluate impact |
| Production schedule changed | Check business impact |
| Safety incident on similar asset | Escalate if related |
| Parts now available | Move to ready backlog |
| Resource constraints | May need to escalate or defer |

### Priority Decision Matrix

Use this matrix to validate or adjust work order priority:

| Factor | P1-P2 (High) | P3-P4 (Medium) | P5-P6 (Low) |
|--------|--------------|----------------|-------------|
| **Safety Impact** | Immediate risk to personnel | Potential risk if not addressed | No safety impact |
| **Production Impact** | Line/plant down or imminent | Reduced capacity/efficiency | Minimal or no impact |
| **Environmental** | Active leak/emission | Potential compliance issue | No environmental impact |
| **Asset Condition** | Failed or failing | Degraded performance | Normal wear |
| **Redundancy** | No backup available | Partial redundancy | Full backup in place |
| **Cost of Delay** | >$10K/day | $1K-$10K/day | <$1K/day |

### Priority Change Authorization

| Change Type | Authorized By |
|-------------|---------------|
| Upgrade to P1/P2 | Supervisor + Ops Manager |
| P3 â†’ P4 or P4 â†’ P3 | Planner |
| Downgrade to P5/P6 | Planner + Requestor approval |
| Close without work | Supervisor |

### Re-assessment Checklist

- [ ] Is the original problem still valid?
- [ ] Has equipment condition changed?
- [ ] Are there related WOs that should be combined?
- [ ] Is the estimated labor still accurate?
- [ ] Are all parts/materials identified and available?
- [ ] Is contractor support still needed?
- [ ] Does Operations still agree with priority?
- [ ] Should this be converted to a PM task?

## 8. Best Practices for Healthy Backlog

### The 2-4 Weeks Rule

A healthy maintenance backlog represents **2-4 weeks** of ready work. This provides:

| Benefit | Explanation |
|---------|-------------|
| **Flexibility** | Enough work to fill schedule gaps |
| **Efficiency** | Time to plan, kit, and optimize |
| **Responsiveness** | Not so much that urgent work gets lost |
| **Visibility** | Manageable size to review regularly |

### Backlog Health Indicators

| Healthy Backlog | Unhealthy Backlog |
|-----------------|-------------------|
| 2-4 weeks of ready work | >6 weeks or <1 week |
| >80% ready to schedule | <60% ready |
| <20% older than 30 days | >40% older than 30 days |
| Clear priority distribution | Everything is "high priority" |
| Regular weekly reviews | No regular review process |
| Declining or stable trend | Growing every week |
| Mix of trades balanced | One trade severely overloaded |

### Best Practices Checklist

#### Daily
- [ ] Review new work requests within 24 hours
- [ ] Update WO statuses as work progresses
- [ ] Flag emergency/urgent work immediately

#### Weekly
- [ ] Conduct backlog review meeting
- [ ] Review all WOs >30 days old
- [ ] Update parts status on waiting WOs
- [ ] Close completed work promptly
- [ ] Generate and review KPI metrics

#### Monthly
- [ ] Deep-dive aging WOs >60 days
- [ ] Review backlog by trade for balance
- [ ] Analyze input vs output trends
- [ ] Adjust capacity planning if needed
- [ ] Report KPIs to management

#### Quarterly
- [ ] Full backlog cleanup exercise
- [ ] Review screening criteria effectiveness
- [ ] Assess priority accuracy
- [ ] Benchmark against targets
- [ ] Update procedures as needed

## 9. Common Problems & Solutions

| Problem | Symptoms | Root Cause | Solution |
|---------|----------|------------|----------|
| **Backlog keeps growing** | WO count increases weekly, backlog weeks >6 | Input exceeds output; insufficient resources or poor screening | Implement stricter screening; add resources; schedule OT blitz; contractor support |
| **Too many "high priority" WOs** | >50% marked P1-P2; everything is urgent | No priority discipline; fear of work not getting done | Re-establish priority criteria; require justification for P1-P2; operations sign-off |
| **Ancient WOs (>90 days)** | Significant % of backlog over 90 days old | No regular review; fear of closing WOs; parts issues | Mandatory monthly cleanup; supervisor authority to close; escalate chronic blockers |
| **Low ready backlog %** | <60% ready to schedule; lots of waiting WOs | Poor parts identification; procurement delays; incomplete planning | Improve job plans; dedicated parts expeditor; planner training; better CMMS use |
| **Duplicate work orders** | Same issue submitted multiple times | No duplicate check; poor CMMS search; lack of feedback to requestors | Screening process; train requestors; CMMS duplicate detection; communicate WO status |
| **Work completed but WOs open** | Technicians don't close WOs; backlog inflated | No accountability; cumbersome close-out; no time allocated | Simplify close-out; mandate same-day closure; supervisor follow-up; mobile CMMS |
| **Imbalanced by trade** | One trade at 8 weeks, another at 1 week | Skill mix mismatch; training gaps; cross-training lacking | Cross-train technicians; contractor support for bottleneck trade; adjust hiring |
| **Reactive work dominates** | >40% emergency/urgent; can't get to planned work | Poor PM program; aging equipment; deferred maintenance | Improve PM effectiveness; address bad actors; make time for planned work |
| **WOs lack information** | Can't schedule due to missing details | Poor work request process; no standards; requestor training gap | Implement request standards; require fields in CMMS; planner follow-up process |
| **No one owns backlog management** | Metrics not tracked; no review meetings; chaos | Unclear roles; competing priorities; no accountability | Assign planner ownership; scheduled reviews; management support; KPI targets |
